使用GitHub+hexo搭建博客

发表于 2016-02-25   |   分类于 2016-02

本博客采用github+hexo+七牛图床建立，刚搭建的时候查了很多资料，走了不少弯路。现在我写下这篇教程，希望帮助大家搭建属于自己的blog。

安装环境：

linux
发行版为opensuse42.1

1.安装node.js hexo git

去node.js官网下载node.js,我选择的版本是5.6。
下载后，解压文件到主目录，然后终端中使用ln命令设置全局
ln -s /home/node-v5.6.0-linux-x64/bin/node /usr/local/bin/node
ln -s /home/node-v5.6.0-linux-x64/bin/npm /usr/local/bin/npm
至此，node.js安装完毕。（详细教程参考顽Shi的Blog）

接下来安装hexo，终端里输入：
npm install hexo-cli -g
等待一小会，hexo就安装完毕了。
再安装一个插件叫hexo-deployer-git,用于推送博客
npm install hexo-deployer-git --save

接下来安装git，终端输入
sudo zypper install git
然后
git -v
查看是否安装成功。（请根据自己的发行版安装）

2.配置hexo

安装完上述环境后，我们先在本地搭建博客。
首先 cd到node.js的bin文件夹下，输入
hexo init blog
这一步会初始化hexo，并生成blog目录。
cd到blog目录下，输入
npm install
然后目录中会出现 _config.yml文件,这个是站点配置文件，打开它。
# site下边的是blog题头，将language字段改为language: zh-Hans设置博客为中文。
# Directory是对应文件目录（无特殊需求不用改）。
# Extensions下边是主题。
# Deployment下是推送设置，这个等到下边推送的时候再说。
在下边添加几行（可选）

since: 2016
avatar: /images/avatar.jpg
social:  
  github: (url)

since是博客创建日期，会显示在底部，avatar是头像，socal是社交链接。

写完之后，终端中输入
hexo g
hexo server
生成静态目录并且本地测试
在浏览器中输入地址 0.0.0.0:4000就可以看到效果了。

2.安装主题

如果你觉得hexo默认的主题不好看，网络上有很多可供你选择，这里我选择的NexT，其源代码在github上。
NexT主题
更多主题看这个：
知乎 hexo主题推荐
具体安装在其页面有详细的介绍，这里不再阐述。

3.写文章

如果你要新建一篇文章，在命令行输入：
hexo n xxx
然后用编辑器打开，文件在blog/source/_posts文件夹下，xxx.md,语法使用的是markdown，写法很简洁，具体可以参考这篇文章Markdown——入门指南 by Te_Lee

每次写完，都要生成静态文件：
hexo g

4.github配置

去github建立一个帐号,验证邮箱（不验证无法创建），然后创建新项目

项目名为xxx.github.io,其中xxx为你的github ID。

先设定git
在终端中输入
git config --global user.name "xxx" xxx为你的id
git config --global user.email xxxx xxxx为你的邮箱

然后我们创建ssh-rsa
在终端中输入
ssh-keygen -t rsa
连续三次次按enter，全部选择默认。
创建后，会在.ssh目录下生成两个文件id_rsa.pub和id_rsa，注意，.ssh目录默认隐藏。
用编辑器打开id_rsa.pub,里边是ssh密钥。

在github上打开项目，选择setting，在右边选择Delopy keys，选择Add delopy key,出现这个界面。

在key里边把id_rsa.pub的密钥全部拷贝过来，title可以不写。

注意: 下边的Allow write access一定要选上，这个将允许修改文件，我刚开始被这个坑了好久TAT。

点击ADD key完成添加，github端设置完成。

####为什么不选择gitcafe？
本人比较懒，gitcafe上边添加delopy key只能是只读的，不能读写。

然后打开站点配置文件
在# Deployment下边修改推送设置

deploy:
  type: git
  repo: git@github.com:xxx/xxx.github.com.git

xxx是你的github ID，默认推送分支为master
修改完后保存，然后终端中生成静态文件。
hexo g
推送。
hexo d
或者使用简便写法
hexo g -d
等待推送完成，在浏览器中访问xxx.github.io即可访问，国外的网站，有点慢。

markdown语法插入图片需要外链，这里推荐七牛，注册帐号后有1G空间，验正身份后10G。

大功告成，enjoy your blog！
